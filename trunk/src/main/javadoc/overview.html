Nujan: Pure Java NetCDF4 and HDF5 writer

Nujan is an open source pure Java software system for writing files in NetCDF4 and HDF5 formats. The acronyms are:
Nujan     NNEW/UCAR Java Authoring of NetCDF4/HDF5     www.ral.ucar.edu/~steves/nujan.html
NNEW     NextGen Network Enabled Weather     www.ral.ucar.edu/projects/nnew
UCAR     University Corporation for Atmospheric Research     www.ucar.edu
NetCDF     Network Common Data Form     www.unidata.ucar.edu
HDF     Hierarchical Data Format     www.hdfgroup.org

NetCDF is a file format primarily used for meteorological and oceanographic data. The NetCDF4 format is built on top of a subset of HDF5. The NetCDF format is maintained by Unidata. Unidata's C, C++, and Fortran implementations of NetCDF are available at Unidata NetCDF, and the Unidata Java reader for NetCDF is available at Unidata NetCDF Java.

HDF5 is a file format primarily used for gridded scientific and engineering data. The HDF5 format is maintained by the HDF Group, which currently has C, C++, and Fortran APIs for HDF5 files. The HDF group also supports a Java JNI (Java Native Interface) to their C code. While the JNI interface does support most of the native HDF5 features, it can be a problem to integrate with other Java packages. Hence the origins of Nujan.

Supported features in Nujan

Nujan supports most of the commonly used features of NetCDF4 and HDF5. The primary differences between the Nujan writer and other NetCDF or HDF5 writers are:
  • Nujan is pure Java, and does not depend on any C code. So Nujan does not use the JNI (Java Native Interface).
  • The Nujan writer only writes new files. It does not update existing files.
  • The Nujan writer writes all the data for a given variable in a single method call. It is not possible to write a variable incrementally.

Notes on Nujan Restrictions

Partial writing
Nujan does not support writing a variable's data incrementally. In Nujan, all of a variable's data must be written in a single call.
Variables over 2 GB total size
Nujan does not support a single variable having a data size exceeding 2 GB. However the total file size can exceed 2 GB.
Other compression methods
Nujan supports the gzip compression method, but does not support szip.
Arbitrary filters
Nujan does not support processing filters other than gzip compression. For example, Nujan does not support checksum filters or custom compression filters.
Modify existing files
Nujan cannot modify existing files; it can only create new files.
Modify after endDefine()
After the client calls endDefine(), Nujan cannot modify the definitions of groups, variables, or attributes.
