Notes on features

Restrict to the NetCDF4 classic model

The NetCDF4 C API offers a flag, CLASSIC_MODEL, that enforces that all features the client uses are within the "classic" subset of NetCDF4. The classic subset is shown in the "NetCDF3" column in the table above. Restricting to the classic model allows a wider variety of readers to handle the file.

Partial writing

NetCDF4 allows writing a variable's data incrementally instead of all at once. Nujan does not support this.

Datasets (files) over 2 GB total size.

This requires two mods to Nujan:
  • Internally, a more complex implementation of chunking. Currently Nujan uses a simple algorithm for chunking.
  • Support for partial writing.

Compression methods other than gzip, such as szip.

Arbitrary filters, such as checksums, custom compression, etc.

Any kind of modifications to existing files:

  • Add/delete/update groups, variables, attributes
  • Change data values

Any kind of modifications after calling endDefine():

  • Add/delete/update groups, variables, attributes

Support for any HDF version other than 1.8.4.

Concurrent reading of a file with an active writer.

Multidimensional coordinate variables.

This feature is seldom used, but occasionally is seen in a lat-lon grid for a non-linear projection.
